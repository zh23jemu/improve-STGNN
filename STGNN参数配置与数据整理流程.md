## 3. 节点特征与预测目标

### 3.1 输入特征

按以下顺序构造每年节点特征矩阵 `X_t ∈ R^(30 x 7)`：

1. `DE`
2. `GDP`
3. `IS`
4. `EDU`
5. `URBAN`
6. `DENSITY`
7. `OPEN`

### 3.2 预测目标

目标变量为：

- `Y_t = EE`

即每个时间点的标签向量为 `30 x 1`。

## 4. 综合邻接矩阵构造

每一年构造一个综合空间图：

`W_t = 0.40 * G_t + 0.35 * E_t + 0.25 * D_t`

其中：

- `G_t` 为地理邻接矩阵
- `E_t` 为经济联系矩阵
- `D_t` 为数字经济相似矩阵

处理要求：

1. 先分别读取同一年 `G_t、E_t、D_t`
2. 检查 3 个矩阵的行列顺序是否完全一致
3. 按上式线性加权
4. 加权后再进行一次行归一化
5. 对角线保留为 `0`

如果模型含自适应邻接矩阵，则使用：

`A_t = W_t + 0.05 * A_adp`

其中：

- `A_adp` 为可训练自适应邻接矩阵
- 自适应部分权重设为 `0.05`

说明：

- 如果 `3.xlsx` 已经是按“滞后一期 DE”构造好的数字经济矩阵，则训练时直接使用，不要再次手工滞后。

## 5. 数据预处理流程

### 5.1 清洗

1. 删除 `jd0.xlsx / 2023` 中最后一行空值记录
2. 检查每个年份是否都恰好有 `30` 个省份
3. 检查 `id` 是否为 `1-30`
4. 检查省份名称映射后是否能与邻接矩阵完全对齐

### 5.2 变换

为了减弱偏态和量纲差异，先做如下变换：

- `GDP -> log1p(GDP)`
- `DENSITY -> log1p(DENSITY)`
- `OPEN -> log1p(OPEN)`

其余变量保持原值：

- `DE`
- `IS`
- `EDU`
- `URBAN`
- `EE`

### 5.3 标准化

标准化必须只用训练集统计量：

1. 用训练集全部时间点、全部省份计算每个输入变量的均值和标准差
2. 用训练集的 `EE` 计算目标变量均值和标准差
3. 验证集和测试集直接使用训练集统计量变换

推荐标准化公式：

`z = (x - mean_train) / std_train`

## 6. 滑动窗口样本构造

### 6.1 时间窗口

采用：

- 历史窗口长度 `Tin = 3`
- 预测步长 `Tout = 1`

即：

- 输入 `t, t+1, t+2` 三年的特征与图
- 预测 `t+3` 年的 `EE`

### 6.2 单个样本形式

对任意起始年 `t`，构造：

- 输入特征序列：`[X_t, X_(t+1), X_(t+2)]`
- 输入图序列：`[W_t, W_(t+1), W_(t+2)]`
- 预测目标：`Y_(t+3)`

因此，单个监督样本形式为：

`([X_t, X_(t+1), X_(t+2)], [W_t, W_(t+1), W_(t+2)]) -> Y_(t+3)`

### 6.3 可生成的全部样本

由于共有 `2011-2023` 共 `13` 年，窗口长度为 `3`，预测步长为 `1`，所以一共可以生成 `10` 个样本：

| 样本编号 | 输入年份 | 目标年份 |
| --- | --- | --- |
| 1 | `2011-2013` | `2014` |
| 2 | `2012-2014` | `2015` |
| 3 | `2013-2015` | `2016` |
| 4 | `2014-2016` | `2017` |
| 5 | `2015-2017` | `2018` |
| 6 | `2016-2018` | `2019` |
| 7 | `2017-2019` | `2020` |
| 8 | `2018-2020` | `2021` |
| 9 | `2019-2021` | `2022` |
| 10 | `2020-2022` | `2023` |

## 7. 训练、验证、测试划分

建议按“目标年份”划分，而不是按原始年份硬切：

| 集合 | 目标年份 | 对应样本 |
| --- | --- | --- |
| 训练集 | `2014-2019` | 样本 `1-6` |
| 验证集 | `2020-2021` | 样本 `7-8` |
| 测试集 | `2022-2023` | 样本 `9-10` |

这样划分的优点：

- 训练样本数相对更多
- 验证集和测试集仍然保持严格时间后移
- 不会把未来年份的信息泄露给过去

## 9. 为什么推荐主配置

你的数据属于典型小样本时空面板：

- 节点数只有 `30`
- 年份只有 `13`
- 滑窗后总样本只有 `10`

因此不宜把模型做得太深。主推荐配置做了以下收缩：

- `GRU` 从 `2` 层降到 `1` 层，降低过拟合风险
- `dropout` 从 `0.20` 降到 `0.15`，减少过强正则带来的欠拟合
- 学习率从 `1e-3` 降到 `3e-4`，提高小样本训练稳定性
- 权重衰减从 `1e-4` 降到 `1e-5`，避免参数被压得过狠
- 自适应邻接权重从 `0.10` 降到 `0.05`，让先验空间结构主导、可学习图只做微调
- `HuberLoss` 比 `MSELoss` 对异常点更稳

## 10. 推荐训练顺序

建议按下面顺序跑，而不是一次性大范围乱调：

### 第一步：先跑基准图权重

先固定其余参数，只比较三组图权重：

| 方案 | 权重设置 |
| --- | --- |
| A | `0.33, 0.33, 0.34` |
| B | `0.40, 0.35, 0.25` |
| C | `0.35, 0.25, 0.40` |

验证集 `R²` 最好的方案进入下一步。

### 第二步：调时间模块复杂度

只比较：

- `GRU layers = 1`
- `GRU layers = 2`

如果 `2` 层在验证集上没有稳定提升，就固定 `1` 层。

### 第三步：调正则与学习率

依次比较：

- `dropout = 0.10 / 0.15 / 0.20`
- `lr = 1e-3 / 5e-4 / 3e-4`

### 第四步：固定最优配置，多随机种子复现实验

建议随机种子至少跑 5 次：

- `42`
- `52`
- `62`
- `72`
- `82`

最终报告：

- 平均 `R²`
- 平均 `RMSE`
- 平均 `MAE`
- 最优单次结果

## 11. 结果评估规则

所有评估指标都应先把预测值从标准化空间反变换回原始 `EE` 尺度，再计算：

- `R²`
- `RMSE`
- `MAE`

不建议直接在标准化后的数值上汇报指标。

## 12. 可直接照着实现的训练流程

1. 读取 `1.xlsx、2.xlsx、3.xlsx、jd0.xlsx`
2. 删除 `jd0.xlsx / 2023` 最后一行空记录
3. 将节点表省份名映射到统一简称
4. 按邻接矩阵顺序重排每年节点数据
5. 提取 `DE、GDP、IS、EDU、URBAN、DENSITY、OPEN` 作为 `X_t`
6. 提取 `EE` 作为 `Y_t`
7. 对 `GDP、DENSITY、OPEN` 做 `log1p`
8. 对每年构造 `W_t = 0.40G_t + 0.35E_t + 0.25D_t`
9. 对 `W_t` 再做一次行归一化
10. 按 `Tin=3, Tout=1` 生成 10 个监督样本
11. 按目标年份划分训练、验证、测试
12. 只用训练集统计量做标准化
13. 用主推荐参数训练 STGNN
14. 用验证集选最优超参数和最优 epoch
15. 在测试集上报告反标准化后的 `R²、RMSE、MAE`
